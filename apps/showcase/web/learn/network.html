The network — block by block

One module, HexfieldNet, does all the reading: a stem conv and norm, the trunk C C A C C A C A — five convolution blocks, three attention blocks — a final norm, and nine heads. Every block runs at width 192 (12 fiber slots × 16 orbit channels) over the variable-length node set of the support; there is no padded grid anywhere. This page walks the trunk block by block: convolutions that see five cells down the line and stop at blockers, attention with one head per win axis, a register lane that can count, and the heads that turn the stream into moves and a verdict — 627,343 parameters in all. The constraints that shape every layer are on the symmetry page; this one is about what the layers do.

01the trunk

The stem is a tied 7-tap convolution from the 46 typed input planes to the 192-channel fiber, then group-norm and ReLU — scalar planes may only write slot-constant weight columns, axis triples lift along the coset structure, and the whole weight is a free parameter passed through a Reynolds projection onto the constraint-satisfying subspace. Its output is the (N, 192) trunk stream, structured as 12 slots × 16, and from there the eight trunk blocks alternate convolution and attention in the fixed pattern CCACCACA.

Two invariants hold through the whole trunk: width 192 everywhere (12 slots × 16 orbit channels, the regular fiber of the symmetry page), and the node mask is re-applied after every parameter-carrying op, so the zero rows that pad a batch to a common length never leak into live nodes.

02ray-tap convolutions

Each C block is a residual block of two convolutions with group-norm after each, ReLU between and after, and a LayerScale (init 1e-4) on the residual branch. The convolutions are ray-tap convs: the center tap reads the cell itself, as in a plain hex conv, but each of the 6 direction taps reads a visibility-masked, distance-weighted sum of up to 5 cells down that direction:

tapd(i)[c]  =  Σk=1..5  α[k, c] · [k ≤ reachside(c)(i, d)] · x(i + k·d)[c]

reach is the ray length of the featurizer’s 12-slot raylen wire for the matching side, axis, and direction: the walk sees up to and including an enemy blocker — the blocker itself is visible — and never past the support edge. The 6 aggregates plus the center tap then enter the same 7-tap GEMM as a plain hex conv, so every conv layer has a 5-cell, blocker-aware receptive field along all three win axes in both directions.

α is a (5 × 16) parameter per conv — per-distance, per-orbit-channel — shared across all 6 directions (the symmetry constraint requires the sharing: a rotation maps one direction tap onto another, so a per-direction α could not satisfy the tie) and broadcast over the 12 slots. Sides ride the channel split: within each 16-channel orbit, the first 8 channels use own-side visibility and the last 8 opp-side. The split lives on the orbit index because symmetries fix orbit channels while permuting slots — a slot split would break the regular-representation structure of the fiber. Cost: ~80 extra parameters per conv, and only the 10 trunk convs carry ray taps — the stem and the head convs are plain 7-tap.

03attention blocks

The A blocks are pre-norm transformer blocks over the joint sequence [ 6 tokens ; N cells ] — the six global summary tokens ride in the same sequence as the board cells. Three structural choices distinguish them from a stock transformer: the 3 heads × 64 partition is pinned one-per-win-axis by the coset structure (the symmetry page derives why no other head count builds), every score gets a relative-position bias gathered from a jointly-tied 237-row table, and padded keys are removed by additive masking before the softmax. A GELU MLP at 2× width follows, with LayerScale on both branches.

04the register lane

Softmax attention computes weighted averages, which cannot represent “how many”: three simultaneous live-4 threats and one look alike through a softmax. So at the exit of every C block, a register refresh updates the 6 tokens from the cells with sigmoid gates and an unnormalized sum:

tokens += out_proj( (Σcells sigmoid(q·kT/√64 + gate_bias) · v) · sum_scale )

Each token learns a pattern query q against cell keys; the sigmoid-gated sum literally counts matching cells. Below, both mechanisms are computed live from their formulas with a fixed illustrative pattern-match query (q·kT/√64 = 6.0 on pattern cells, 0.0 on background; v carries the match signal): the gated sum grows by the same amount for every added threat, while the softmax baseline saturates after the first one.

The lane’s numbers: gate_bias init −2.5, so a token’s background gate starts near 0.08 rather than integrating the whole board; sum_scale init 1/32 (learned) keeps updates O(1)–O(10) at typical match counts; out_proj starts near zero, so the lane is a numeric no-op at initialization while its gradients stay live. The token stream is carried in fp32 on the half-precision serve path so late-block count increments survive rounding — and because the final LayerNorm normalizes magnitudes away, the scalar heads also read the pre-final-norm token mean as one extra input block: the raw count magnitudes reach the value estimate directly.

05the heads

Nine heads read the trunk stream. The four per-cell heads each run a plain 7-tap conv, a ×2 tied expansion, then group-pool the 12 slots away and apply a linear on the resulting 32-wide invariant read — one output per node. The five scalar heads each read 64-wide group-pooled ×4 blocks: their assigned tokens, the masked mean of all cells, and the pre-final-norm token mean.

Inference computes policy + value (+ moves_left when the search requests it); the other six heads exist to shape the trunk during training — auxiliary targets that force the stream to carry the opponent’s reply, per-cell values, and short-horizon estimates it would otherwise be free to forget.

06the parameter budget

An untied trunk of the same width would store ~4.3 M trunk parameters alone. The ×12 tie yields a ~7× smaller model that computes like a full 192-channel network — the dense weights exist at forward time, generated from the free blocks.
