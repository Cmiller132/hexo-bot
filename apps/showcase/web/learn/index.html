hexfield — how it works

hexfield is an AlphaZero-family agent for Hexo, a Connect6-family game played on an unbounded hexagonal grid. One neural network reads a position and returns a policy (a distribution over legal moves) and a value (who is winning); a Gumbel Monte-Carlo tree search uses that network to look ahead and returns a move stronger than the raw policy; and the network is trained entirely on its own games to imitate what the search found. There is no human game database in the loop — only an optional behavioral-cloning warm start before it begins. This page is the map; the three chapters walk each part.

01the loop

Everything below runs as one repeating cycle, one epoch at a time. Self-play generates games with the current network; the trainer samples recent games and nudges the network toward the search’s conclusions; the improved network is checkpointed and served back to self-play. Evaluation branches off the loop — in this codebase its verdict is informational: it measures strength against a pool of frozen opponents but does not gate or roll back a checkpoint.

03why this is unusual

The board has no edges, so the input has no shape. Hexo is played on an unbounded hexagonal grid: there is no 19×19 plane to convolve over, and a game can wander anywhere. Instead of cropping a fixed window, hexfield builds a support set — exactly the cells that matter right now (stones, legal moves, and a one-cell halo) — and feeds the network a variable-size graph that grows and shrinks with the game. Cells are ordered legal-first, which gives the legal-prefix property: the policy head’s first legal_count outputs are exactly the legal moves, so the network can never score an illegal one. How the support set is built →

Global attention over an irregular point set. Convolutions on this graph are direction-typed — the neighbour along each of the six hex axes gets its own weights, because line direction is the entire point of a connection game — but they only spread information one ring per block. Interleaved attention blocks give every cell a view of the whole position in a single step: all-pairs attention over the support cells plus 8 learned summary tokens, with a relative-position bias keyed on the exact hex offset between two cells (a 217-offset disk, then ring buckets). A threat forming on the far side of the board is visible immediately, not after ten convolutions. How attention handles hex geometry →

Gumbel-Top-m instead of Dirichlet noise. Classic AlphaZero explores at the root by stirring Dirichlet noise into the prior and forcing playouts; hexfield removed those knobs entirely. It draws m = 32 root candidates by adding Gumbel noise to the policy logits, then runs Sequential Halving so the visit budget concentrates on survivors — with a guarantee that the played move improves on the raw policy even at small budgets. The improved root distribution is exactly what the network trains toward (policy_target = "gumbel"), and threat-space search force-includes tactical cells so the candidate draw can never miss a win-in-one.
