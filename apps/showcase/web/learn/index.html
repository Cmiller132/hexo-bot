hexfield_eq — how it works

hexfield_eq is an AlphaZero-family agent for Hexo with one defining property: the network is exactly equivariant under all 12 symmetries of the hexagonal lattice. Rotate or reflect the board and the policy rotates with it and the value is unchanged — not statistically, but by construction, for every position and every weight the net will ever have. One 627k-parameter network reads a position and returns a policy (a distribution over legal moves) and a value (who is winning); a Gumbel Monte-Carlo tree search uses that network to look ahead; and the network trains entirely on its own games. These pages explain each part, with the interactive figures built from the same code and data the production system runs.

01the loop

Everything runs as one repeating cycle, one epoch at a time. Self-play generates games with the current network; every searched decision writes one training row; the trainer samples a window of recent rows and nudges the network toward the search’s conclusions; the improved network is checkpointed and served back to self-play. Evaluation runs outside the loop and observes — it measures strength against a fixed roster and never gates, promotes, or halts training.

The parameter count is the point, not the punchline: an untied trunk of the same width would store ~4.3 M trunk parameters alone. The 12-fold weight tie yields a ~7× smaller model that computes like a full 192-channel network — dense weights exist at forward time, generated from the free parameter blocks.

03properties of the design

Exact symmetry, zero augmentation. For every lattice symmetry g and every position, f(g·board) = g·f(board) — the policy over the transformed board is exactly the transformed policy, the value exactly unchanged. Every layer type carries a constraint that makes this hold: convolutions keep 84 free weight blocks of a nominal 1008, attention heads are pinned one-per-win-axis, and the position-bias table collapses to 81 free values per block. The training pipeline applies no symmetry augmentation, and no parameter is spent representing a rotated copy of a pattern another parameter already knows. How the constraint works →

The board the net sees is built per position. There is no padded grid: a support set is rebuilt every position, ordered legal-first so the policy head’s first legal_count outputs are the legal moves — the mask is a slice that cannot drift out of sync. Each cell carries 46 feature planes (graded line-and-openness readings of all 18 length-6 windows through it) and 12 blocker-aware ray lengths. What the network reads →

Convolutions that see down the line. Each trunk convolution’s six direction taps read a visibility-masked, distance-weighted span of up to 5 cells down each direction — sight ends at the support edge and at (but including) an enemy blocker. Every conv layer has a full window-length receptive field along all three win axes, for ~80 extra parameters per conv. The ray-tap convolution →

A lane that can count. Softmax attention computes weighted averages, so three simultaneous live-4 threats and one look alike. Six summary tokens are updated at every conv block’s exit by sigmoid-gated unnormalized sums — a mechanism that literally counts matching cells — and the raw count magnitudes reach the value head directly. Search that improves the policy, not just the move. Root selection is Gumbel-Top-m (m = 16) with Sequential Halving; the recorded training target is the improved policy π′, not the visit histogram (under Sequential Halving, visit counts are a schedule artifact). No Dirichlet noise, no forced playouts anywhere. How the search works →
