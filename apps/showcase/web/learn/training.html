The training — targets, losses, and how strength is measured

The trainer’s whole job is to make the network agree with its own search. Every searched decision in self-play writes one training row — the facts of the position plus everything the search concluded about it. Each epoch, a tapered window of recent rows is expanded back into tensors and the network takes AdamW steps against nine loss heads summed with fixed weights. Strength is measured outside the loop: paired seat-swapped games against a fixed roster, scored pentanomially, feeding a rolling Bradley–Terry pool that observes but never gates.

01one row per decision

A training row stores facts, not tensors. The position is recorded as its placement history and rebuilt at training time — the 46 feature planes are never stored, so a featurizer fix or retune applies retroactively to every row ever written. Around those facts sit the targets: the search’s improved policy π′, the game outcome, short-horizon value averages, the opponent’s reply, and a measure of how much the search disagreed with the raw network.

At training time rows are expanded back into tensors in Rust (expand_backend = "rust"): rebuild the support from the placement history, recompute all 46 planes and 12 ray lengths, and project the policy targets onto the legal prefix. No symmetry transform is applied — the network is equivariant by construction, so a rotated copy of a position teaches it nothing new.

02the replay window

Each epoch trains on up to 56,000 rows drawn from a window over recent games. The window is not fixed-size: it grows sublinearly with the total number of rows the run has ever generated — a power-law taper with exponent 0.65 and a 7,000-row floor — takes newest shards first, and down-samples uniformly toward a 210,000-row keep target. Early in a run the net trains on almost everything it has; later, the window keeps growing but ever more slowly, balancing freshness against variety.

Two details keep this machinery honest. Determinism: shard ordering derives from game keys, not file timestamps, so a resumed run selects exactly the same window — the producing epoch is parsed out of the key itself. The reuse governor: a counter persisted in the checkpoint credits 8 training-row-units for every newly generated self-play row and debits the rows actually trained; when the buffer goes stale, the trainer throttles rather than grinding over the same lagging data — it cannot over-fit a buffer that self-play has stopped refreshing.

03nine heads, one sum

The total loss is a fixed-weight sum over nine heads. Policy heads learn the search’s conclusions; the value family learns outcomes at four time scales; two auxiliary heads (opponent reply, per-cell Q) shape the trunk’s representation. Scalar targets are never regressed directly — every scalar head predicts a distribution over 65 bins on [−1, 1].

Structural legality. The policy CE is computed over each row’s legal prefix directly — the support ordering makes the first legal_count outputs be the legal moves, so there is no mask to apply. Target mass off the prefix is a hard error that raises, not a masked zero: a target disagreeing with the legality slice means the row and the rebuilt position have drifted, and the trainer refuses to average over it.

Two-hot targets. A scalar target (outcome, short-term value, moves-left, cell Q) becomes a distribution: its mass is split between the two adjacent bin centers in proportion to proximity, and the head trains with plain CE. At decode time the prediction is read back as the softmax expectation over bin centers — the encode–decode round trip is exact, and the head is free to express uncertainty as spread rather than being forced to a point estimate.

04optimization

The optimizer recipe is compact, with one genuinely unusual entry: the gradient clip is adaptive. The tied trunk concentrates gradient — each conv’s 84 free weight blocks receive gradient through every one of the gathers that assemble the dense kernel — so its global gradient norm runs hotter than an untied net’s, and a fixed clip threshold tuned on untied-net intuition would silently miscalibrate. Instead the clip tracks 1.75× an EMA of the pre-clip norm (a static 5.0 while the EMA warms up), following the net’s own scale.

Batches of 256 rows never run as one tensor: positions vary in size, so the batch is split into micro-buckets under a B·Npad² budget — a few fat buckets of similar-sized positions instead of one padded monolith. Losses across buckets share step-global denominators, so the optimizer step’s gradient is mathematically independent of how the batch was split (up to floating-point reassociation). The run itself warm-starts from a 2,400-step behavioral-cloning prefit — the same architecture briefly trained to imitate a fixed game corpus, loaded weights-only with a fresh optimizer — skipping the slowest opening-competence epochs of self-play.

05paired games, pentanomial scoring

Strength is measured directly, every 5 epochs, from a 192-game budget. Games run at 512 visits, greedy after an 8-ply temperature-1.0 opening. The measurement problem is that shared openings correlate results — so the unit of statistics is not the game but the pair: both games share one sampled opening line (the leader searches its opening, the follower replays it) and swap seats, so a pair differs only by who holds which side. Hexo has no draws, so a completed pair scores 0, 1, or 2 candidate wins; a pair with one truncated game scores 0.5 or 1.5. Results land in a five-bucket (“pentanomial”) histogram whose pair-level variance absorbs the opening-induced correlation — treating the two games as independent would understate the variance and quote confidence intervals narrower than the truth.

06the roster, and reading the numbers

All results feed a rolling Bradley–Terry pool anchored at Strix = 0 Elo. The candidate-vs-champion verdict is single-epoch by construction — roughly ±130 Elo of standard error — and it never tightens: it is a gross-regression tripwire, not a progress meter. The fixed-anchor curves (Strix, SealBot, the brackets — the same label every epoch) pool information across epochs, and they are the actual measure of progress.

Evaluation cannot gate, promote, or halt training — this is asserted in the eval path, not merely left as convention. The one feedback loop is the head audit: every epoch the moves-left head is scored against recent games (rank-correlation and near-end error gates); on failure, the search lever that consumes it is disabled until the head recovers. The model keeps training either way — only the search stops trusting a head that has gone bad.
