    Play your bot against shrimp

    The match API lets your own Hexo bot play rated matches against any checkpoint on this server, over plain HTTP — no browser, no cookies. Matches land in the games feed and your agent name is ranked on the ELO leaderboard, exactly like a human player.

    ↓ bot-api.md — full protocol ↓ hexo_match_client.py — Python SDK
    the .md file is the complete wire-protocol reference; the SDK is a single zero-dependency Python file — copy it into your project.

    Quickstart — three steps

    1 Pick an opponent: GET /api/bots lists playable checkpoints and the allowed sims budgets.
    2 Subclass the SDK's BotAdapter — one method returns your stone.
    3 Run it. The SDK handles polling, retries, and the two-stones-per-turn shape.
    from hexo_match_client import BotAdapter, MatchServer

class MyBot(BotAdapter):
    def select_stone(self, state):
        # state["history"]: every move so far   [{q, r, color}, ...]
        # state["legal"]:   cells you may play  [{q, r}, ...]
        # state["you"]:     your color (0 moves first)
        cell = my_engine.think(state)
        return cell["q"], cell["r"]

result = MyBot().play_match(
    MatchServer("https://<this-server>"),
    agent="my-bot",            # your public leaderboard name
    checkpoint_id="main7-ep90", sims=128, agent_color="random",
)
print(result["result"])        # {"winner": ..., "termination": ...}
    demo without writing a bot: python hexo_match_client.py --server <url> --agent test --checkpoint main7-ep90 --sims 64 --random-demo

    Endpoints

    callwhat it does
    POST /api/match create a match → {match_id, token, state}; the token is your auth, shown once
    GET /api/match/{id}?wait=25 state; long-polls up to 25s until you can act (bearer token)
    POST /api/match/{id}/move place ONE stone {q, r} — post-opening turns are two calls
    POST /api/match/{id}/resign concede
    POST /api/match/{id}/retry re-run the server bot's turn after status == "bot_failed"
    every call after create carries Authorization: Bearer <token>

    The state payload

    {
  "status": "your_turn",          // your_turn | bot_thinking | bot_failed | finished
  "you": 0,                       // your color (0 moves first)
  "to_move": 0,
  "stones_left_this_turn": 2,     // turns place two stones after the opening
  "history": [{"q":0,"r":0,"color":0}, ...],   // whole game, move order
  "legal":   [{"q":1,"r":-1}, ...],            // cells you may play NOW
  "result":  null,                // set once finished; human_result +1 = you won
  "bot": {"checkpoint_id":"main7-ep90","sims":128, ...}
}
    rules in one line: unbounded hex grid, six in a row wins, color 0's opening stone is auto-played at the origin, then two stones per turn. legal is authoritative — never compute legality yourself.

    Limits & manners

    Matches share the human-game budget per IP: active-match caps and move/create rate limits (HTTP 429 with a readable detail). A match that goes silent for the idle_timeout_s reported at create (default 10 min) is recorded as a timeout — long thinks are fine, vanishing is not. Your matches run on the same worker pool as live human games, so keep concurrent matches to a handful.
